mt — User ManualA keyboard-driven terminal workspace launcher

Overview #

mt is a keyboard-driven TUI (terminal UI) that turns your most-used commands into a fast, navigable menu. No mouse. No memorizing flags. Navigate a three-pane interface, press Enter, see output inline.

Common uses:

  • Launch macOS apps without leaving the terminal
  • Run Docker compose, git, and dev-server commands from a single interface
  • Trigger multi-step deploy pipelines as a sequence — Build → Test → Deploy
  • Run multiple commands in parallel with Space + Enter
  • Execute commands on remote servers over SSH

Installation #

go install

Requires Go 1.21 or later.

go install github.com/mrwalker511/mt@latest

This places the mt binary in $GOPATH/bin (usually ~/go/bin). Make sure that directory is on your $PATH.

Build from source

git clone https://github.com/mrwalker511/mt
cd mt
make build   # produces ./mt in the project root

Other make targets:

CommandAction
make buildCompile binary to ./mt
make testRun test suite
make vetRun go vet
make cleanRemove compiled binary

Quick Start #

Run mt from any directory:

mt

If no config file is found, built-in defaults are shown (see Default Workspaces).

3-pane layout

┌────────────────┬────────────────┬────────────────────────────────┐ │ Domains │ Targets │ Output / Status │ │ │ │ │ │ > App Launch │ > Microsoft │ Press [Enter] to launch │ │ Dev Tools │ Notes │ │ │ Infrastructure│ Safari │ │ │ Context/Git │ Terminal │ │ │ │ │ │ │ │ │ │ └────────────────┴────────────────┴────────────────────────────────┘ [←/h focus left] [→/l focus right] [? help]
  • Left pane — Domain list. Arrow keys or j/k move the cursor. Domains group related targets (e.g. "Outlook", "Infrastructure", "Context/Git").
  • Middle pane — Target list for the selected domain. Press Enter to run, Space to toggle multi-select.
  • Right pane — Command output, live status header, and hints. Scroll with j/k when focused here.

First steps

  1. Press → or l to move focus to the middle pane.
  2. Use ↑/↓ or k/j to select a target.
  3. Press Enter to run it. Output appears in the right pane.
  4. Press ? at any time for a help overlay.
  5. Press q or Ctrl+C to quit.

Configuration #

File locations

mt searches for a config file in this order, stopping at the first match:

  1. System config dir (platform-dependent):
    • macOS  ~/Library/Application Support/mt/config.yaml
    • Linux  ~/.config/mt/config.yaml
  2. Local override — ./mt.yaml in the current working directory

If neither exists, built-in defaults are used. To create your own config:

# macOS
mkdir -p ~/Library/Application\ Support/mt
cp mt.yaml.example ~/Library/Application\ Support/mt/config.yaml

# Linux
mkdir -p ~/.config/mt
cp mt.yaml.example ~/.config/mt/config.yaml
Local override caution A ./mt.yaml in the current directory overrides the system config. Only run mt inside directories you trust. Config files must not be world-writable — mt will refuse to load them if they are.

YAML structure

# Single-workspace config (simplest form)
domains:
  - name: "My Project"
    targets:
      - name: "Start Server"
        status: "Press [Enter] to start"
        cmd: ["npm", "run", "dev"]
      - name: "Run Tests"
        cmd: ["npm", "test"]
# Multi-workspace config
workspaces:
  - name: "Microsoft"
    domains:
      - name: "Outlook"
        targets:
          - name: "New Email"
            cmd: ["open", "ms-outlook://compose"]
            launch_msg: "Opening new email…"
  - name: "Dev"
    domains:
      - name: "Context/Git"
        targets:
          - name: "Git Status"
            cmd: ["git", "status"]

Field reference

FieldRequiredDescription
nameYesDisplay label for workspace, domain, or target
cmdNoCommand and arguments to execute on Enter, as a YAML list: ["git", "status"]. Omit to make a target display-only.
statusNoHint text shown in the right pane before the command has been run. Supports newlines.
launch_msgNoMessage shown when cmd exits successfully with no output (e.g. open -a produces no stdout).
hostNoSSH host in [user@]hostname form. If set, cmd is executed as ssh host <cmd>. See SSH targets.
sequenceNoOrdered list of target names to run in series. If any step fails, the sequence stops. See Sequence.
appsNomacOS only. List of app names; each expands to an open -a target. Shorthand for common launchers. See Apps shorthand.
includeNoList of YAML file paths to merge into this config. See Include directive.

Multi-workspace

Wrap your domains in a workspaces: block to switch between independent contexts with Tab / Shift+Tab. Each workspace has its own domains, targets, run history, and output cache.

workspaces:
  - name: "Work"
    domains:
      - name: "Apps"
        apps: ["Microsoft Outlook", "Microsoft Teams"]
      - name: "Dev"
        targets:
          - name: "Build"
            cmd: ["make", "build"]

  - name: "Personal"
    domains:
      - name: "Apps"
        apps: ["Mail", "Safari", "Spotify"]

Workspace switching resets cursor positions and clears multi-select. Per-target output is preserved per-workspace.

Apps shorthand macOS

The apps: key under a domain accepts a list of macOS app names. Each entry auto-expands to a full target that runs open -a <name>:

domains:
  - name: "Apps"
    apps:
      - "Microsoft Outlook"
      - "Microsoft Edge"
      - "Slack"
      - "Visual Studio Code"

Explicit targets: entries within the same domain come first; apps: expansions are appended after them.

Include directive

Split a large config into multiple files using include::

# ~/.config/mt/config.yaml
include:
  - work.yaml
  - ~/shared/team-tools.yaml

domains:
  - name: "Personal"
    targets:
      - name: "Check Mail"
        cmd: ["open", "-a", "Mail"]

Include rules:

  • Relative paths resolve relative to the file that declares them.
  • All paths (after resolving symlinks) must remain within your home directory ($HOME). Paths outside $HOME are rejected as a security measure.
  • Circular includes are detected and silently skipped.
  • Included domains: are merged; included workspaces: are appended.

Features #

Single command

The default mode. Navigate to any target with a cmd: field and press Enter. The command runs asynchronously; the right pane shows output when it completes. While a command runs, a visual indicator shows it is in progress and Enter is ignored.

Output is bounded to 1 MB. Longer output is automatically truncated with a notice.

Sequence

A target with a sequence: field executes named targets in order. If any step exits non-zero, the sequence stops immediately.

domains:
  - name: "CI"
    targets:
      - name: "Full Pipeline"
        sequence: ["Build", "Test", "Deploy"]

      - name: "Build"
        cmd: ["make", "build"]
      - name: "Test"
        cmd: ["make", "test"]
      - name: "Deploy"
        cmd: ["./scripts/deploy.sh", "production"]

Output from all completed steps is concatenated and shown in the right pane when the sequence finishes (or fails). A sequence step that has no cmd: field produces an error and stops the sequence.

Multi-select parallel

Run multiple targets at the same time:

  1. Focus the middle pane (→ or l).
  2. Press Space to toggle each target you want to run. A check mark appears next to selected targets.
  3. Press Enter to run all selected targets in parallel.

When all commands complete, output is shown with === Target Name === headers so you can identify each result. Selection is cleared automatically after execution.

SSH targets

Set the host: field to run a command on a remote server:

domains:
  - name: "Production"
    targets:
      - name: "Deploy"
        host: "deploy@prod.example.com"
        cmd: ["./deploy.sh", "--env", "production"]
        launch_msg: "Deploy started…"

      - name: "Tail Logs"
        host: "app@prod.example.com"
        cmd: ["journalctl", "-fu", "myapp"]

With host: set, mt runs ssh host <cmd>. SSH uses your existing key configuration (~/.ssh/config, ssh-agent). No credentials are stored by mt.

Host validation The host: value must match [a-zA-Z0-9][a-zA-Z0-9._@%-]*. Values starting with - or containing spaces are rejected at config load time to prevent SSH option injection.

Live status polling

When the active workspace contains certain named domains, mt polls for live status every 5 seconds and shows it in the right-pane header:

Domain nameWhat is polledDisplay
Context/Git git rev-parse --abbrev-ref HEAD
git status --porcelain		 Current branch + modified file count
Infrastructure docker ps --filter name=postgres
docker ps --filter name=redis		 Container status for postgres and redis

Polling only runs for domains present in the current workspace and stops when you quit (all in-flight probes are cancelled). Each probe has a 3-second timeout — on a slow machine or when Docker Desktop is stopped, polls complete quickly rather than blocking the UI.

Output management

After a command runs, its output is shown in the right pane. Per-target output is cached in memory so navigating away and returning restores the previous result.

KeyAction
↑/k (right pane)Scroll up
↓/j (right pane)Scroll down
cClear output from the right pane
yCopy output to clipboard
SSave output to ~/.mt/logs/mt-YYYYMMDD-HHMMSS.txt

Cached output per target is bounded at 256 KB. Log files are written with exclusive creation (O_EXCL) to prevent race conditions.

Config hot-reload

Press R to reload your config file without quitting. This lets you edit mt.yaml in a separate terminal and instantly see the changes in the running mt session.

Hot-reload resets cursor positions, clears selections, and clears cached output. If the new config is invalid, an error is shown and the previous config continues in use.

AI assistant

Press / to open the AI prompt bar at the bottom of the screen. Type a natural-language request and press Enter. The local LLM maps your query to one of three actions:

LLM responseWhat happens
RUN:<target_name>Runs the named target from the current workspace (same as navigating to it and pressing Enter)
CMD:["prog","arg",…]Executes the returned command array directly — useful for dynamically generated osascript or shell commands
INFO:<text>Displays a short text answer in the right pane

Example requests:

  • open outlook → runs the Open Outlook target
  • create a 30-min meeting with john@company.com tomorrow at 2pm → generates and runs an osascript that creates the event in Outlook
  • what does git stash do → returns a short explanation
  • start docker and check postgres → runs Docker Up then Postgres if they exist as targets

Press Esc to cancel the input or cancel an in-flight request. Requires a local LLM server — see LLM configuration.

LLM configuration #

The AI assistant supports three providers: LiteLLM (local proxy, default), llama.cpp (direct server), and OpenAI (cloud). All three speak the OpenAI /v1/chat/completions API. Switch with a single provider: line.

LiteLLM proxy (default — port 4000)

LiteLLM acts as a unified gateway in front of any model backend (Ollama, vLLM, cloud APIs, etc.).

pip install litellm

# Start the proxy with any backend model
litellm --model ollama/llama3.1:8b
# or
litellm --model gpt-4o-mini   # requires OPENAI_API_KEY in env

llama.cpp server (port 8080)

# Build or download a llama.cpp release, then:
./llama-server -m /path/to/model.gguf --port 8080
llm:
  provider: llamacpp
  base_url: http://localhost:8080   # default; omit if using the default port

OpenAI / OpenAI-compatible cloud

llm:
  provider: openai
  model: gpt-4o-mini        # or gpt-4o, o1-mini, etc.
  base_url: https://api.openai.com   # override for vLLM, LMStudio, etc.
  api_key: sk-...           # or set OPENAI_API_KEY env var

FieldDefaultDescription
providerlitellmlitellm, llamacpp, or openai.
modelprovider-specificModel name passed to the provider. LiteLLM uses its own routing (e.g. ollama/llama3.1:8b); llama.cpp ignores it.
base_urlhttp://localhost:4000 / :8080 / openai.comAPI base URL. Override to point at any OpenAI-compatible server.
api_keyenv: OPENAI_API_KEYRequired for openai provider only. Ignored by LiteLLM and llama.cpp.

Microsoft Copilot

For Microsoft Office tasks, Copilot is available directly within Outlook, Word, and Excel if your Microsoft 365 subscription includes it. The built-in Copilot Chat target in the Microsoft Edge domain opens copilot.microsoft.com. The Copilot in Word and Copilot in Excel targets in their respective domains provide status notes reminding you to use Copilot from within the app.

Programmatic Microsoft Copilot API integration (for use directly from the mt AI prompt) requires Azure credentials and is outside the scope of this tool's built-in LLM feature.

Security Model #

mt is a local single-user tool. Its trust boundary is the config file: whoever can write the config can run arbitrary commands as the user who runs mt. The security controls below harden that boundary.

SSH host validation

The host: field is validated at config load time against the pattern ^[a-zA-Z0-9][a-zA-Z0-9._@%-]*$. This rejects values starting with -, which SSH would interpret as option flags — the vector for ProxyCommand injection. A config containing an invalid host is rejected entirely and mt falls back to built-in defaults with an error message.

World-writable config rejection

Before reading any config file (including files loaded via include:), mt checks the file permissions. If the world-writable bit (0002) is set, mt refuses to load the file. On a shared or multi-user system this prevents another user from injecting commands by modifying a config file they can write to.

If mt rejects your config with a "world-writable" error, fix it with:

chmod o-w ~/.config/mt/config.yaml

Include path restrictions

Include paths (after resolving symlinks) must resolve within your home directory ($HOME). This prevents a malicious project's ./mt.yaml from reading sensitive YAML files elsewhere on disk (e.g. ~/.kube/config, /etc/secrets.yaml). Symlinks that point outside $HOME are also rejected.

Circular include protection

Include chains are tracked by canonical path. If a file has already been loaded in the current chain it is silently skipped, preventing infinite loops.

Command execution safety

Commands are always passed as argument arrays, never through a shell. cmd: ["git", "status"] calls exec.Command("git", "status"), not sh -c "git status". This means shell metacharacters (;, |, &, etc.) in arguments are always treated as literals, not shell syntax.

Output and cache bounds

Command output is truncated at 1 MB before being stored or displayed. Per-target cached output is limited to 256 KB. This prevents runaway commands from exhausting available memory.

Log file safe creation

Log files are created with O_EXCL (exclusive creation), which fails if anything — including a symlink — already exists at the target path. This closes a TOCTOU race that could otherwise be used to overwrite an arbitrary file.

Log files are stored in ~/.mt/logs/ with mode 0640 (readable only by owner and group). The directory itself is created with mode 0750.

Goroutine and process cancellation

All commands and polling operations use a shared context.Context. Pressing q or Ctrl+C cancels the context, which immediately terminates any in-flight exec.Command processes. No goroutines are left running after mt exits.

Troubleshooting #

Config errors

Error messageCauseFix
"file is world-writable" Config file has 0002 permission bit set chmod o-w <config-file>
"invalid ssh host" host: value contains illegal characters or starts with - Use format user@hostname or plain hostname
"path must be within home directory" include: path resolves outside $HOME or via a symlink that escapes $HOME Move the included file inside your home directory
"no domains or workspaces defined" Config file is valid YAML but has no content Add at least one domain with one target
"Config error — using built-in defaults" Config file was found but could not be parsed Check YAML syntax; run cat config.yaml | python3 -c "import sys,yaml;yaml.safe_load(sys.stdin)"

Clipboard not working

The y key copies output using a platform clipboard tool:

  • macOS Uses pbcopy (built-in, always available)
  • Linux Tries xclip, then xsel, then wl-copy (Wayland)

On Linux, install one of these if copying doesn't work:

# X11
sudo apt install xclip
# or
sudo apt install xsel

# Wayland
sudo apt install wl-clipboard

SSH commands fail immediately

mt does not handle SSH authentication. Ensure:

  • Your SSH key is loaded: ssh-add ~/.ssh/id_ed25519
  • The host is reachable: ssh user@hostname echo ok
  • SSH config in ~/.ssh/config has the right settings for the host

Commands appear to hang

Some commands block waiting for stdin (e.g. interactive prompts). Since mt captures output without a TTY, the command may hang. Use non-interactive flags (e.g. docker exec -it → docker exec, git commit with -m). Quit mt (q) to cancel any in-flight command.

Terminal is too narrow

mt requires a minimum terminal width to render correctly. Widen your terminal window if you see a "terminal too narrow" message.

Hot-reload wipes run history

Pressing R resets all cursor positions and cached target output. This is intentional — the workspace structure may have changed. Run your commands again after reloading.
